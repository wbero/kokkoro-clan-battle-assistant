# 轴文件标准格式

本文档说明可可萝自动会战助手当前支持的两种轴文件：顺序轴和开关轴。轴文件建议保存为 UTF-8 编码的 `.txt` 文件，然后通过 App 主界面的“导入轴文件”按钮导入。

## 如何选择轴类型

| 类型 | 适合场景 | 核心行为 |
|---|---|---|
| 顺序轴 | 按顺序安排角色 UB、AUTO、提示和一次性状态调整 | 每个动作进入队列，前一个动作完成后才执行下一个动作 |
| 开关轴 | 在指定时间或触发条件后，把五个角色 SET 和 AUTO 调整为完整目标状态 | 每个节点声明完整状态，程序根据画面执行最少点击并确认状态 |

最重要的区别是：

- 顺序轴中的 `点击=角色N` 表示一次完整的“SET 开启 → 等待角色 UB → SET 关闭”生命周期。
- 开关轴中的 `SET=... | AUTO=...` 表示持续目标状态，不会在角色 UB 后自动恢复。

## 通用书写规则

### 文件编码和注释

- 使用 UTF-8 编码。
- 空行会被忽略。
- `#` 开头的整行是注释。
- 字段使用半角等号 `=`。
- 同一节点的字段使用半角竖线 `|` 分隔。
- 多个角色或 SET 状态使用半角逗号 `,` 分隔。

```text
# 这是注释
1:20 | 点击=角色2 | 提示=执行角色2
```

### 文件头

| 字段 | 是否必填 | 允许值 | 说明 |
|---|---:|---|---|
| `轴类型` | 标准格式必填 | `顺序`、`开关` | 决定解析和执行方式 |
| `轴名称` | 否 | 任意非空文字 | 不填写时使用导入文件名 |
| `点击间隔` | 否 | `1`～`5000` | 单位毫秒，默认 `100`；主要用于顺序轴动作 |
| `角色1`～`角色5` | 否 | 角色别名 | 仅作为顺序轴 `点击=` 的别名 |

角色位置始终按照战斗界面从左到右排列：

```text
角色1, 角色2, 角色3, 角色4, 角色5
```

角色别名示例：

```text
角色3=原晶
```

之后下面两种写法等价：

```text
1:20 | 点击=角色3
1:20 | 点击=原晶
```

别名只适用于顺序轴的 `点击=`。`UB后=`和`卡帧=`必须使用 `角色1`～`角色5`，不能使用别名。

### 时间格式

时间必须写成 `M:SS`，有效范围是 `0:00`～`1:30`。

有效示例：

```text
1:30
1:05
0:59
0:08
```

无效示例：

```text
01:30
1:5
1:31
```

建议按照倒计时从大到小排列。时钟发生跳秒时，已经跨过的节点会按照文件中的原始顺序进入等待队列。

## 通用触发条件

顺序轴和开关轴支持相同的四种节点触发方式。

### 定时触发

不填写额外触发字段时，节点在时间到达后执行：

```text
1:20 | ...
```

### 指定角色 UB 后触发

节点到时后开始监听，只接受节点进入等待状态之后出现的下一次对应角色 UB：

```text
1:20 | UB后=角色3 | ...
```

如果角色3在节点进入等待之前已经释放 UB，不会满足这个节点。

### Boss UB 后延迟触发

```text
0:30 | UB后=BOSS | 延迟=1.20 | ...
```

`延迟`单位为秒，必须大于 `0` 且不超过 `30` 秒。

当前版本没有可靠的 Boss 专属 UB 画面信号，因此实际语义是：节点到时后开始计算最短延迟，延迟结束且控制画面可信时执行。它不是对 Boss UB 的视觉确认。

### 卡帧触发

```text
1:10 | 卡帧=角色4 | ...
```

节点到时后，悬浮窗进入手动逐帧状态：

1. 点击“下一帧”观察动作。
2. 到达目标帧后点击“确定：角色4”。
3. 程序解除软暂停并点击角色4。
4. 继续执行本节点剩余动作。

同一节点不能同时填写 `UB后`和`卡帧`。

## 顺序轴标准格式

### 完整模板

```text
# 顺序轴标准模板
轴类型=顺序
轴名称=顺序轴示例
点击间隔=100

角色1=角色A
角色2=角色B
角色3=角色C
角色4=角色D
角色5=角色E

[轴]
1:30 | SET=关,关,关,关,关 | AUTO=开 | 提示=设置开局状态
1:20 | 点击=角色1
1:15 | 点击=角色2,角色4 | 提示=角色2后接角色4
1:10 | UB后=角色3 | 点击=角色2 | 提示=角色3 UB后执行角色2
1:00 | 卡帧=角色4 | 提示=确认角色4动作帧
0:45 | UB后=BOSS | 延迟=1.20 | 点击=AUTO
0:30 | AUTO=关
0:20 | SET=开,关,开,关,关
0:10 | 提示=即将结束
```

顺序轴必须包含 `[轴]` 段。`[轴]`上方是文件头，下方是时间节点。

可直接使用的示例文件：[sequence-axis-standard.txt](./examples/sequence-axis-standard.txt)。

### 顺序轴动作字段

| 字段 | 格式 | 行为 |
|---|---|---|
| `点击` | `点击=角色1` | 执行角色的一次性 SET 生命周期 |
| `点击` | `点击=角色2,角色4` | 按从左到右的书写顺序串行执行 |
| `点击` | `点击=AUTO` | 切换一次 AUTO，并等待画面确认 |
| `点击` | `点击=BOSS` | 当前只显示 Boss 提示，不执行自动确认 |
| `AUTO` | `AUTO=开`、`AUTO=关` | 把 AUTO 调整到指定状态 |
| `SET` | `SET=开,关,开,关,关` | 把五个角色 SET 调整到指定状态 |
| `提示` | `提示=文字` | 在悬浮窗显示提示，不执行控制点击 |

`SET`必须包含五个值，依次对应角色1～角色5，而且每个值只能是 `开`或`关`。

### `点击=角色N`的执行过程

```text
确保该角色 SET 开启
→ 等待该角色能量从满值下降，确认已经释放 UB
→ 确保该角色 SET 关闭
→ 继续下一个动作
```

因此下面的节点不是同时点击两个角色：

```text
1:15 | 点击=角色2,角色4
```

实际顺序是：

```text
角色2 SET开 → 角色2 UB → 角色2 SET关
角色4 SET开 → 角色4 UB → 角色4 SET关
```

### 顺序轴开局状态

建议在 `1:30`声明开局 SET 和 AUTO：

```text
1:30 | SET=关,关,关,关,关 | AUTO=开
```

开局最多声明一个 `SET`目标和一个 `AUTO`目标。程序会先确认开局状态，再开始普通顺序节点。

### 顺序轴卡帧注意事项

`卡帧=角色4`本身已经包含确认时在暂停菜单点击角色4，并在返回战斗后自动进入“等待角色4 UB → 关闭角色4 SET”的生命周期：

```text
1:00 | 卡帧=角色4
```

如果再写一次 `点击=角色4`：

```text
1:00 | 卡帧=角色4 | 点击=角色4
```

确认卡帧时会点击角色4，返回战斗后继续等待角色4 UB并关闭 SET；同一行再次写 `点击=角色4` 会被视为重复目标并去重。其他角色的 `点击=` 动作仍按原顺序执行。

## 开关轴标准格式

### 完整模板

```text
# 开关轴标准模板
轴类型=开关
轴名称=开关轴示例

[轴开局] | SET=关,关,关,关,开 | AUTO=开 | 提示=设置开局状态
1:12 | UB后=角色5 | SET=关,关,关,关,关 | AUTO=开 | 提示=角色5 UB后全关
0:57 | SET=开,关,开,开,开 | AUTO=开 | 提示=定时切换
0:30 | UB后=BOSS | 延迟=1.20 | SET=关,开,关,关,开 | AUTO=开
0:18 | 卡帧=角色3 | SET=开,关,开,关,开 | AUTO=开 | 提示=确认角色3动作帧
```

开关轴不使用 `[轴]`。它必须包含且只能包含一个 `[轴开局]`。

兼容旧格式时，只要存在 `[轴开局]`，即使省略 `轴类型=开关`也会识别为开关轴；标准文件仍建议明确填写轴类型。

现有卡帧示例：[switch-axis-pause-frame-test.txt](./examples/switch-axis-pause-frame-test.txt)。

### 开关轴目标字段

每个开关轴节点都必须包含：

```text
SET=五个开关状态 | AUTO=开或关
```

例如：

```text
SET=关,开,关,关,开 | AUTO=开
```

表示：

| 控制项 | 目标状态 |
|---|---|
| 角色1 SET | 关 |
| 角色2 SET | 开 |
| 角色3 SET | 关 |
| 角色4 SET | 关 |
| 角色5 SET | 开 |
| AUTO | 开 |

程序会读取画面中的实际状态，只点击与目标不一致的控制项，并在画面确认全部状态后才进入下一个节点。

### 开关轴节点类型

#### 开局节点

```text
[轴开局] | SET=关,关,关,关,开 | AUTO=开 | 提示=开局
```

- 必须且只能有一个。
- 必须包含完整五角色 `SET`和 `AUTO`。
- 在时钟 `1:30`～`1:28`且控制画面可信时开始收敛。

#### 定时节点

```text
0:57 | SET=开,关,开,开,开 | AUTO=开
```

时钟到达或越过该时间后开始调整目标状态。

#### 角色 UB 后节点

```text
1:12 | UB后=角色5 | SET=关,关,关,关,关 | AUTO=开
```

节点到时后等待角色5下一次 UB，然后收敛完整目标状态。

#### Boss 延迟节点

```text
0:30 | UB后=BOSS | 延迟=1.20 | SET=关,开,关,关,开 | AUTO=开
```

节点到时后等待至少 `1.20`秒，再收敛完整目标状态。

#### 卡帧节点

```text
0:18 | 卡帧=角色3 | SET=开,关,开,关,开 | AUTO=开
```

确认卡帧时先点击角色3，然后将所有 SET 和 AUTO 调整到本节点声明的目标状态。卡帧节点也必须包含完整目标。

## 顺序轴和开关轴对照

| 行为 | 顺序轴 | 开关轴 |
|---|---|---|
| 段标记 | `[轴]` | `[轴开局]` |
| 开局状态 | 推荐在 `1:30`填写 | `[轴开局]`强制填写 |
| 节点目标 | 可以只写一个动作 | 每个节点必须写完整 SET 和 AUTO |
| 角色点击 | 一次性 SET 生命周期 | 仅卡帧确认时直接点击指定角色 |
| `提示` | 支持 | 支持 |
| `UB后=角色N` | 触发本行后续动作 | 触发完整状态收敛 |
| `UB后=BOSS` | 延迟后执行本行动作 | 延迟后收敛完整状态 |
| `卡帧=角色N` | 确认点击角色，再执行剩余动作 | 确认点击角色，再收敛完整状态 |
| 角色别名 | `点击=`支持 | 不支持 |

## 如何导入和验证轴文件

1. 将轴文件保存为 UTF-8 `.txt`文件。
2. 打开 App，点击“导入轴文件”。
3. 选择文件。
4. 在轴列表中检查类型和节点数量。
5. 点击有效轴，将其设为当前轴。

导入成功后会显示类似：

```text
顺序轴示例 [SEQUENCE]
开关轴示例 [SWITCH]
```

无效轴仍会出现在列表中，但不能被选中，并会显示校验原因。战斗开始后轴选择会锁定，必须重置战斗后才能更换。

## 常见错误

| 错误 | 原因 | 修改方法 |
|---|---|---|
| `轴类型必须为顺序或开关` | `轴类型`填写了其他内容 | 改成 `轴类型=顺序`或`轴类型=开关` |
| `缺少 [轴] 段` | 顺序轴没有 `[轴]` | 在文件头和时间节点之间增加 `[轴]` |
| `开关轴缺少 [轴开局]` | 开关轴没有开局节点 | 增加一个完整的 `[轴开局]` |
| `开关轴只能包含一个 [轴开局]` | 写了多个开局节点 | 只保留一个 |
| `点击间隔必须在 1..5000 毫秒` | 点击间隔为0、负数或过大 | 改为 `1`～`5000` |
| `时间格式必须为 M:SS` | 分钟或秒格式错误 | 使用 `1:05`、`0:30`等格式 |
| `时间必须在 0:00..1:30` | 时间超出战斗范围 | 调整到 `0`～`90`秒 |
| `SET 必须包含五个开或关` | SET数量不是5或出现其他文字 | 填写五个 `开/关`值 |
| `AUTO 只能为开或关` | AUTO使用了其他值 | 改为 `AUTO=开`或`AUTO=关` |
| `未知角色或别名` | 顺序轴 `点击=`角色不存在 | 使用角色1～角色5或已声明别名 |
| `UB后必须指定角色1到角色5` | UB角色名无效 | 使用 `角色1`～`角色5`或`BOSS` |
| `Boss UB后节点必须声明延迟` | `UB后=BOSS`缺少延迟 | 增加 `延迟=1.20`等字段 |
| `Boss延迟必须大于0且不超过30秒` | 延迟范围错误 | 使用 `0`到`30`之间的正数 |
| `同一节点不能同时声明UB后和卡帧` | 两种触发方式同时出现 | 只保留一种触发方式 |
| `开关轴节点必须包含五个 SET 状态和 AUTO 状态` | 开关节点目标不完整 | 同时填写完整 `SET`和`AUTO` |

## 执行安全与当前限制

- 未启用本 App 的无障碍服务时，只会识别和显示计划，不会执行点击。
- 勾选“只识别，不执行点击”后，不会产生真实点击。
- 控制按钮或菜单画面不可信时，程序会等待或进入安全暂停，不会猜测点击。
- 角色 UB 使用对应角色能量条从满值下降作为证据；没有识别到证据时不会继续后续动作。
- `UB后=BOSS`目前使用节点到时后的最短延迟，不是 Boss UB 视觉识别。
- 顺序轴中的 `点击=BOSS`目前只产生提示，不执行 Boss 专属点击或确认。
