# 顺序轴行为说明

顺序轴本阶段仅记录需求，不进入实现范围。

## 核心语义

- `点击=角色N`不是永久切换，也不是简单地把当前状态取反。
- 节点触发后，工具确保角色N的 SET/立即发动状态为开。
- 工具等待该角色实际释放一次 UB。
- UB完成后，工具关闭该角色的 SET，使这次设置只生效一次。
- 完成上述生命周期后，才进入下一个顺序点击动作。

同一时间包含多个角色时，必须保留原文顺序；每个角色都具有独立的“一次性 SET -> UB -> 关闭 SET”生命周期。AUTO点击仍按轴中出现顺序执行。

## 尚未设计的部分

- 角色连续 UB 时，多个一次性 SET 生命周期如何并行或串行；
- Boss UB 后节点的可靠触发；
- 角色 UB 与 Boss UB 连续冻结计时器时的归属判断；
- 顺序轴卡帧节点与一次性 SET 生命周期的结合；
- UB结束的最终视觉/能量判据。

在这些问题形成独立设计并通过真机数据验证前，Android版本不宣称完整支持顺序轴。
